# AI Sketch to CAD Confirmation Workflow Design

## 背景

用户希望做一个 Web 端产品：上传手绘草稿图后，AI 先分析草图内容，再通过苏格拉底式追问让用户澄清场景、尺寸、用途、比例、材料、几何约束和行业语义。系统在信息足够后生成可预览的 demo，用户可以继续确认或修改，直到最终确认后导出可编辑的 DXF CAD 文件。

这个产品不应被设计成“一次上传图片，AI 直接生成最终 CAD”。更稳妥的定位是“AI 草图转 CAD 的确认式建模平台”：AI 负责识别、推断、提问和生成草案；用户负责确认关键语义；系统用结构化中间模型驱动预览和 DXF 导出。

## 产品定位

产品采用“通用平台 + 场景约束包”的设计。

通用平台适用于多个行业场景，负责上传、视觉识别、候选场景判断、问题生成、回答管理、可视化预览、版本迭代和 DXF 导出。

场景约束包负责行业语义。第一版优先实现“市政/管线/道路沿线外业草图”场景包，但整体架构必须允许后续扩展到户型装修、柜体家具、机械零件、电气布线等场景。

## 第一版范围

第一版 MVP 支持：

- 上传一张手绘草图图片。
- AI 识别草图中的线条、框、文字、尺寸、箭头、符号、节点和相对位置。
- AI 判断候选场景，并让用户确认。
- 第一版重点支持市政/管线/道路沿线外业草图。
- AI 按场景包生成澄清问题。
- 问题支持可变数量选项，不固定为 A/B/C/D。
- 每个选择题都提供“其他，请输入”。
- 数值、尺寸、坐标、比例类问题直接使用输入框，并要求单位或校验单位。
- 澄清完成后生成可预览的矢量 demo。
- 用户可以基于 demo 继续修改和确认。
- 最终确认后导出 DXF。

第一版不支持：

- 直接导出 DWG。
- 自动生成施工级完整图纸。
- 同时完整支持所有 CAD 行业。
- 跳过用户确认直接生成最终文件。
- 复杂三维建模、BIM、STEP 或 IFC。

## 核心原则

通用的是流程，不通用的是行业语义。

AI 输出的第一结果不是 CAD，而是结构化中间模型。CAD 由中间模型导出。这样才能保证对象可编辑、问题可追溯、预览可迭代、导出可稳定。

每个追问都必须绑定到一个明确的对象、区域、尺寸、文字、符号或全局约束。系统不应进行空泛聊天。

问题不是越多越好。系统应优先询问会影响 CAD 几何、比例、图层和对象类型的高影响问题。

## 用户流程

1. 用户上传手绘草图。
2. 系统显示原图，并进行初步识别。
3. AI 给出候选场景列表，让用户确认。
4. 用户选择场景或输入其他场景。
5. 系统加载对应场景约束包。
6. AI 提取草图对象并生成初始中间模型。
7. AI 生成一组按优先级排序的问题。
8. 用户逐题回答，也可以跳过并标记为待确认。
9. 系统根据回答更新中间模型。
10. 信息足够后生成可预览 demo。
11. 用户在 demo 上确认、修改或继续回答问题。
12. 系统反复更新 demo，直到用户确认最终结果。
13. 系统从中间模型导出 DXF。

## 场景判断

上传后，AI 先判断候选场景，不直接进入细节追问。

候选场景示例：

- 市政/道路/管线/外业测绘草图
- 建筑/户型/装修平面图
- 柜体/家具定制图
- 机械零件草图
- 电气/弱电布线图
- 其他，请输入

AI 应给出推荐场景和置信度，但必须让用户确认。用户确认后才进入该场景的约束体系。

## 场景约束包

每个场景约束包包含：

- 场景名称和描述。
- 常见对象类型。
- 常见符号类型。
- 常见尺寸和单位规则。
- 常见几何约束。
- 常见图层映射。
- 问题模板。
- 校验规则。
- DXF 导出规则。

第一版市政/管线/道路沿线外业草图约束包包含：

- 线对象：道路边线、道路中心线、管线、水沟、河道、围墙、边界线。
- 点对象：检查井、雨水井、污水井、桩点、节点、接入口、转折点。
- 面对象：建筑物、院落、地块、站点、设施范围。
- 文字对象：地物名称、尺寸标注、施工说明、方向说明。
- 尺寸对象：沿线距离、建筑尺寸、偏移距离、宽度、高度。
- 默认单位候选：米、毫米、厘米。
- 图层候选：道路、管线、水系、建筑、节点、尺寸、文字、待确认。

## 中间模型

中间模型是系统核心。它保存所有可编辑对象、约束、问题、回答、置信度和版本历史。

示例结构：

```json
{
  "project": {
    "scene": "municipal_pipeline_survey",
    "unit": "m",
    "status": "clarifying"
  },
  "entities": [
    {
      "id": "entity_001",
      "type": "building",
      "label": "邮政大厦",
      "geometry": {
        "kind": "rectangle",
        "width": 50,
        "height": 17
      },
      "layer": "building",
      "confidence": 0.72,
      "confirmation": "needs_user_confirmation"
    }
  ],
  "constraints": [
    {
      "id": "constraint_001",
      "type": "distance",
      "value": 50,
      "unit": "m",
      "applies_to": ["entity_001"]
    }
  ],
  "questions": [],
  "answers": [],
  "versions": []
}
```

## 问题生成规则

AI 生成问题时需要区分问题类型。

选择类问题：

- 选项数量由 AI 根据上下文生成，不固定为 4 个。
- 选项应尽可能覆盖常见答案。
- 最后始终提供“其他，请输入”。
- 可提供“不确定/暂时跳过”，并将相关对象标记为待确认。

数值类问题：

- 直接提供输入框。
- 必须询问或显示单位。
- 应做基本校验，例如非负数、合理范围、单位一致性。

尺寸类问题：

- 要说明该尺寸作用于哪个对象或哪两点之间。
- 如果 OCR 识别出多个候选值，应让用户选择或手动修正。

几何关系类问题：

- 选项可以包括平行、垂直、相接、相交、同轴、等距、未知、其他。

语义类问题：

- 询问对象含义，例如“这条线表示道路边线、管线中心线、水沟，还是其他？”

全局约束类问题：

- 询问单位、比例、图纸方向、基准线、导出标准。

## 问题优先级

问题按影响程度排序：

1. 全局场景、单位、比例、基准线。
2. 主要几何对象类型。
3. 决定 CAD 形状的关键尺寸。
4. 冲突尺寸和模糊文字。
5. 图层、材料、用途和施工说明。
6. 低置信度对象和可暂缓确认内容。

系统应避免一次性向用户展示大量问题。第一版可以采用分步问题队列，每次展示一个主要问题，同时允许用户查看待确认列表。

## Demo 预览

预览界面包含：

- 原始手绘图。
- 生成的矢量 demo。
- 对象列表。
- 当前问题。
- 待确认对象列表。
- 图层显示开关。
- 版本记录。

矢量 demo 中：

- 已确认对象使用正常样式。
- 待确认对象使用明显标记。
- 冲突尺寸使用警示标记。
- 用户点击对象后可以查看来源、置信度、关联问题和已确认属性。

demo 不是最终文件，而是用于继续确认的可视化工作区。

## DXF 导出

DXF 必须从中间模型生成，不能直接从图片生成。

导出规则：

- 每类对象映射到对应 CAD 图层。
- 线、矩形、多段线、圆、文字、尺寸标注使用标准 DXF 实体。
- 待确认对象可以选择不导出，或导出到“待确认”图层。
- 导出前显示最终检查清单。
- 用户确认后下载 DXF。

第一版只承诺 DXF 可编辑，不承诺符合所有施工图规范。

## 校验和错误处理

系统需要校验：

- 单位是否一致。
- 关键尺寸是否缺失。
- 相邻尺寸是否冲突。
- 对象是否存在未确认类型。
- 图层是否可映射。
- DXF 导出是否成功。

如果信息不足，系统不应假装完成，而应生成下一轮问题或把对象放入待确认列表。

## 架构模块

前端模块：

- 上传与项目创建。
- 原图查看。
- 问题回答组件。
- 选项和自定义输入组件。
- 数值和单位输入组件。
- 矢量 demo 预览。
- 对象属性编辑。
- 图层面板。
- DXF 下载。

后端模块：

- 图片存储。
- 视觉识别服务。
- 场景分类服务。
- 场景约束包加载器。
- 问题生成器。
- 中间模型管理。
- 校验器。
- demo 渲染数据生成器。
- DXF 导出器。

AI 模块：

- 草图理解。
- OCR 和符号解释。
- 场景候选判断。
- 结构化对象提取。
- 澄清问题生成。
- 回答解释和中间模型更新建议。

规则模块：

- 场景约束包。
- 问题模板。
- 单位和尺寸校验。
- 图层映射。
- DXF 实体映射。

## 可扩展性

新增行业时，不改通用流程，只新增场景约束包和必要的渲染/导出映射。

例如：

- 户型场景包新增墙、门、窗、房间、梁柱、面积、层高。
- 柜体场景包新增板件、孔位、封边、五金、材料、厚度。
- 机械场景包新增孔径、倒角、公差、中心线、视图、材料。
- 电气场景包新增开关、插座、回路、桥架、线缆、配电箱。

如果某个行业需要 DXF 之外的格式，应作为该场景包的扩展能力处理。

## 验收标准

第一版完成后应满足：

- 用户可以上传手绘草图。
- 系统可以给出候选场景，并让用户确认。
- 系统可以加载市政/管线/道路沿线场景包。
- 系统可以生成可变数量选项的问题。
- 系统可以处理“其他，请输入”。
- 系统可以处理数值类和单位类输入。
- 系统可以将用户回答写回中间模型。
- 系统可以生成一个可预览的矢量 demo。
- 系统可以标记待确认对象。
- 用户可以多轮确认并更新 demo。
- 用户最终确认后可以下载 DXF。
- DXF 打开后对象按图层组织，并且主要线、面、文字可编辑。

## 关键产品判断

该产品可以做通用化，但通用化方式不是一套通用 CAD 语义，而是一套通用确认流程叠加多个场景约束包。

第一版应聚焦市政/管线/道路沿线外业草图，因为样本图已经明确指向这个场景，而且该场景很适合通过“基准线、距离、节点、建筑、文字标注”的方式转为 2D DXF。

平台从第一天保留多场景扩展能力，但不在第一版同时实现多个行业的完整能力。
